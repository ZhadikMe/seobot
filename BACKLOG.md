## ✅ Уже сделано

### Базовый pipeline (v1)
- Оценка времени перед стартом (~X–Y мин, N стр. × K яз.)
- Живой прогресс перевода: "🌍 Перевод: N/K страниц..."
- PR через git clone+push (убран лимит 546 файлов)
- Приоритет пользовательского токена над env GITHUB_TOKEN
- Потоковые логи перевода в Railway (-u флаг)
- `fix_title_refresh`, `fix_internal_links` в pipeline
- Антифейк detection (character ratio)
- Body fallback для сайтов без `<main>`/`<article>`
- Повторный прогон на уже обработанном репо (детектор не переизвлекает из архива)
- Merge hreflang при частичном повторном переводе

### v2 (апрель 2026)
- **21 язык** (было 12): добавлены el, uk, ko, zh, ja, sk, fi, ar, hi
- **Кэш nav/footer** — nav/header/footer переводятся 1 раз на сессию (~30% экономии API)
- **detect_source_lang** — автодетект языка сайта из `<html lang="">`, неанглийские сайты работают корректно
- **RTL поддержка** — арабский (ar) получает `dir="rtl"`, LTR-переводы арабского источника теряют rtl
- **Расширенное извлечение текста** — `<span>`, `<figcaption>`, `<input placeholder>`, `.card-title/.card-text`
- **Режим "только перевод"** — пропускает SEO-фиксы, только переводит + PR
- **Режим "SEO без перевода"** — все SEO-шаги без перевода + PR
- **Очередь репозиториев** — до 3 одновременно (1 active + 2 waiting), уведомление о позиции
- **Аудит до/после** — показывает "Проблем было X → стало Y" после завершения исправлений
- **Выбор языков UX** — кнопки "🌍 Все языки (21)" / "✏️ Выбрать вручную" + "Выбрать все" / "Снять все" в сетке
- **Исправлен user_id в очереди** — `state.key.user_id` вместо `message.from_user.id` (был ID бота при callback)
- **Исправлена оценка времени** — учитывает все 21 язык, корректная оценка для seo_only режима
