## ✅ Уже сделано

### v3 (апрель 2026)
- **Пропуск домена для seo_only** — `domain_keyboard()` показывает кнопку "⏭️ Пропустить" для всех режимов (не только seo_only)
- **`fix_external_links()`** — удаляет все внешние `<a href="https://...">` ссылки: текстовые анкоры остаются как текст, теги с изображениями оставляют только img. Ссылки на собственный домен, `mailto:` и `tel:` не трогаются. Заменяет `fix_nofollow`.
- **`fix_twitter_card()`** — копирует `og:title`, `og:description`, `og:image`, `og:url` в соответствующие `twitter:*` теги. Не затрагивает lang-директории и archive-страницы.
- **`fix_cloudflare_stubs()`** — удаляет Cloudflare challenge-страницы (6 паттернов: "Attention Required", "DDoS protection", "Checking your browser" и др.) перед обработкой сайта. Первый шаг в pipeline.

- **`fix_h2()`** — автоматическая вставка H2 для страниц без него: Groq генерирует из H1 + контента, батч-перевод для lang-версий
- **`fix_thin_content()`** — добавляет intro/outro параграфы на архивные страницы с малым текстом (<200 слов), переводит для lang
- **`fix_lang_descriptions()`** — после фикса EN-описаний батч-переводит их для всех 21 языков (1 Groq вызов на язык вместо N×per-page)
- **`_groq_call()`** — централизованный хелпер с exponential backoff (429 → retry), заменяет urllib на requests
- **`_generate_description()`** — теперь пропускает страницы с уже нормальным description (50–155 симв), меньше лишних API вызовов
- **`_count_words()` в audit.py** — `re.findall` по всем `<article>` и суммирование (было `re.search` → только 1-й article); архивные страницы с несколькими постами теперь не ложно помечаются как "тонкие"
- **Апостроф в description** — убрано двойное экранирование `'` → `&#39;` в double-quoted атрибутах

### Базовый pipeline (v1)
- Оценка времени перед стартом (~X–Y мин, N стр. × K яз.)
- Живой прогресс перевода: "🌍 Перевод: N/K страниц..."
- PR через git clone+push (убран лимит 546 файлов)
- Приоритет пользовательского токена над env GITHUB_TOKEN
- Потоковые логи перевода в Railway (-u флаг)
- `fix_title_refresh`, `fix_internal_links` в pipeline
- Антифейк detection (character ratio)
- Body fallback для сайтов без `<main>`/`<article>`
- Повторный прогон на уже обработанном репо (детектор не переизвлекает из архива)
- Merge hreflang при частичном повторном переводе

### v2 (апрель 2026)
- **21 язык** (было 12): добавлены el, uk, ko, zh, ja, sk, fi, ar, hi
- **Кэш nav/footer** — nav/header/footer переводятся 1 раз на сессию (~30% экономии API)
- **detect_source_lang** — автодетект языка сайта из `<html lang="">`, неанглийские сайты работают корректно
- **RTL поддержка** — арабский (ar) получает `dir="rtl"`, LTR-переводы арабского источника теряют rtl
- **Расширенное извлечение текста** — `<span>`, `<figcaption>`, `<input placeholder>`, `.card-title/.card-text`
- **Режим "только перевод"** — пропускает SEO-фиксы, только переводит + PR
- **Режим "SEO без перевода"** — все SEO-шаги без перевода + PR
- **Очередь репозиториев** — до 3 одновременно (1 active + 2 waiting), уведомление о позиции
- **Аудит до/после** — показывает "Проблем было X → стало Y" после завершения исправлений
- **Выбор языков UX** — кнопки "🌍 Все языки (21)" / "✏️ Выбрать вручную" + "Выбрать все" / "Снять все" в сетке
- **Исправлен user_id в очереди** — `state.key.user_id` вместо `message.from_user.id` (был ID бота при callback)
- **Исправлена оценка времени** — учитывает все 21 язык, корректная оценка для seo_only режима
- **Исправлен re-run детектор** — `detector.py` LANGS обновлён до 21 языка (ko, el, uk и др. теперь корректно обнаруживаются при повторном запуске)
- **Исправлен перевод многострочных `<p>`** — `patch_html` нормализует пробелы в паттерне (`\s+`), теперь абзацы с `\n` внутри тега корректно заменяются
- **Исправлен перевод заголовков с `<br/>`** — `H1–H4` с вложенными тегами разбиваются на отдельные текстовые узлы через `_extract_text_nodes`, каждый заменяется независимо
