# SEO Bot — Документация

**Версия:** v4 (апрель 2026)  
**Деплой:** Railway (auto-deploy из GitHub `master`)

---

## Что такое SEO Bot

Telegram-бот для автоматической SEO-оптимизации статических сайтов. Применяет исправления, переводит на 21 язык и открывает Pull Request на GitHub.

Бот рассчитан на **два разных сценария использования**:

**Сценарий A — восстановление из веб-архива.**  
Целевая аудитория: владельцы старых сайтов, которые уже не работают, но сохранились в Wayback Machine. Пользователь отправляет ссылку на архивный снимок — бот сам скачивает сайт, восстанавливает структуру файлов и запускает pipeline. Технического бэкграунда не требуется.

**Сценарий Б — готовый статический сайт на GitHub.**  
Целевая аудитория: разработчики или владельцы, у которых сайт уже лежит в GitHub-репозитории в виде HTML-файлов. Бот клонирует репо, применяет SEO-исправления и открывает Pull Request.

---

## Архитектура

```
bot.py          — Telegram UI, FSM-состояния, оркестрация pipeline
pull.py         — Скачивание сайта из web.archive.org (wget), восстановление структуры
detector.py     — Определяет тип репо (для GitHub-потока)
audit.py        — SEO-аудит: сканирует HTML, формирует список проблем
fixes.py        — Шаги исправлений (in-place)
translate.py    — Перевод через WowAI API, hreflang, языковой switcher
run_local.py    — Локальный запуск pipeline без бота и GitHub
```

**Внешние зависимости:**
- **WowAI Translate API** — машинный перевод (21 язык)
- **Groq API** (llama-3.1-8b-instant) — генерация meta description, H1, H2
- **GitHub API + git** — скачивание репо и создание Pull Request
- **wget** — скачивание сайта из Wayback Machine
- **aiogram 3** — Telegram Bot Framework

## Два сценария запуска

### Сценарий A: ссылка на web.archive.org (основной)

Пользователь отправляет ссылку вида `https://web.archive.org/web/20140606/http://example.com/`.

Бот:
1. Определяет домен и временную метку снимка
2. Считает число страниц через CDX API и показывает оценку времени
3. Скачивает весь сайт через `wget` (один проход, --wait=3 против rate limit)  
   > ⚠️ Бот работает на Railway — скорость скачивания ограничена пропускной способностью сервера и rate limit archive.org. На выделенном сервере или локально процесс был бы быстрее.
4. Восстанавливает ассеты (CSS/JS/картинки) с повторными попытками
5. Исправляет структуру файлов (PHP→HTML, extensionless→HTML, query-string URLs)
6. Запрашивает домен, GitHub-токен, языки → подтверждение
7. Запускает SEO-исправления и перевод
8. Открывает Pull Request

**Оценка времени** до старта (на экране подтверждения):
```
📥 Скачивание: не более 12 мин
⚙️ Восстановление ассетов: ~4 мин
🔧 SEO-исправления: ~2 мин
🌍 Перевод (3 яз.): ~8 мин
⏳ Итого: ~26 мин
```

### Сценарий Б: ссылка на GitHub-репозиторий

Пользователь отправляет `https://github.com/user/sitename`.

Бот клонирует репо через GitHub API, определяет структуру (site/ папка, архивный дамп, HTML в корне) и запускает тот же pipeline без этапа wget.

## Режимы работы

| Режим | Описание |
|---|---|
| 🔍 Только аудит | Сканирует, выводит список SEO-проблем. Ничего не меняет. |
| 🔧 Полный (SEO + перевод) | Все шаги: SEO-фиксы + перевод + Pull Request |
| 🛠️ Только SEO | SEO-фиксы (meta, canonical, schema, og:image...) без перевода |
| 🌍 Только перевод | Пропускает SEO-шаги, только переводит + PR. Для повторного прогона с новыми языками. |

## Шаги исправлений

### Сценарий A (archive.org) — SEO-шаги

| # | Шаг | Что делает |
|---|---|---|
| 1 | `fix_archive_scripts` | Удаляет Wayback Toolbar инъекции из HTML |
| 2 | `fix_canonical` | Добавляет абсолютный `<link rel="canonical">` с реальным доменом |
| 3 | `fix_title_refresh` | Обновляет год в title/description на текущий |
| 4 | `fix_descriptions` | Генерирует уникальный meta description через Groq AI (120–155 символов) |
| 5 | `fix_h1` | Добавляет H1 на страницы без заголовков (Groq) |
| 6 | `fix_schema` | Добавляет BreadcrumbList Schema.org |
| 7 | `fix_og_image` | Добавляет полную OG-разметку: `og:image` (из первого подходящего изображения), `og:title`, `og:url`, `og:type` (`website`), `og:site_name` (из домена), `og:locale` (из `<html lang="">`). Уже существующие теги не перезаписываются. |
| 8 | `fix_external_links` | **Удаляет** все внешние `<a href="https://...">` ссылки; текстовые анкоры остаются как текст. Ссылки на собственный домен, mailto:, tel: не трогаются |
| 9 | `fix_robots_txt` | Дополняет robots.txt: удаляет устаревшую директиву `Host:`, добавляет или обновляет строку `Sitemap:` на актуальный URL. Полная перезапись только если файла нет или он содержит `example.com`-заглушку. |
| 10 | `fix_sitemap` | Генерирует sitemap.xml со всеми страницами и hreflang |

### Сценарий Б (GitHub) — SEO-шаги

Все шаги из сценария A, плюс:

| # | Шаг | Что делает |
|---|---|---|
| + | `fix_cloudflare_stubs` | Удаляет страницы-заглушки Cloudflare (`window.location.reload()`) |
| + | `fix_preloader` | Отключает preloader-анимации (блокируют отрисовку) |
| + | `fix_h2` | Генерирует H2 через Groq для страниц без подзаголовков |
| + | `fix_thin_content` | Добавляет intro/outro параграфы на страницы с <200 слов |
| + | `fix_twitter_card` | Копирует og:title/description/image в twitter:* теги |

### Шаги перевода (оба сценария)

| # | Шаг | Что делает |
|---|---|---|
| 1 | `fix_translations` | Переводит на выбранные языки (с кэшем nav/footer) |
| 2 | `fix_hreflang_translated` | Добавляет hreflang на переведённые страницы |
| 3 | `fix_internal_links` | Исправляет внутренние ссылки в переведённых версиях |
| 4 | `fix_lang_switcher` | Добавляет/обновляет floating dropdown для переключения языков |

## pull.py: скачивание и восстановление

Модуль отвечает за превращение архивного снимка в работающий статический сайт.

**Шаги внутри `pull_snapshot`:**

1. **wget** — один проход: `--mirror --page-requisites --convert-links --wait=3` — скачивает HTML + все ассеты (CSS, JS, картинки) за один запрос к archive.org
2. **Восстановление ассетов** — 60 сек пауза (archive.org rate limit), затем повторные попытки для 404-файлов. ~30% CDX-ссылок требует восстановления
3. **PHP → HTML** — `rename_php_to_html`: переименовывает `.php` → `.html`, переписывает внутренние ссылки
4. **Query-string URLs** — `_fix_wget_param_filenames`: старые WordPress `/?p=123` wget сохраняет как `index.html@p=123` (Windows) или `index.html?p=123` (Linux). Коллизии с `index.html` → переименовываются в `_qs__p=123.html`. `_fix_querystring_links` переписывает все `href="/?p=123"` → `href="/_qs__p=123.html"`
5. **Extensionless HTML** — `rename_extensionless_html`: файлы без расширения, содержащие `<!DOCTYPE html>` → добавляет `.html` *(работает ПОСЛЕ шага 4, иначе двойное расширение)*
6. **Очистка архива** — удаляет мета-файлы wget, дубликаты, временные артефакты

## Перевод: как работает

1. **Детектирование языка** из `<html lang="">` — `detect_source_lang()` проверяет первые 5 HTML-файлов. Исходный язык исключается из целевых переводов.
2. **Кэш nav/footer** — UI-строки переводятся 1 раз на сессию (~30% экономии API-вызовов)
3. **Извлечение текста**: title, description, H1–H4, p, li, button, span, figcaption, input[placeholder], .card-title, .card-text
4. **Батч-перевод** через WowAI API (по 50 сегментов на запрос)
5. **Антифейк-фильтр**: перевод отклоняется если ≥70% символов совпадает с оригиналом (для кириллических/иероглифических — character ratio)
6. **RTL поддержка**: арабский получает `dir="rtl"`, LTR-переводы арабского источника теряют rtl
7. **Запись**: переведённый файл → `/{lang}/path/index.html`
8. **hreflang** мержится в оригинальный файл — не перезаписывает прошлые прогоны

### Повторный запуск (incremental)

Если репозиторий уже был обработан ботом (PR смёрджен):
- Детектор распознаёт существующие языковые папки (`/ru/`, `/ko/` и т.д.)
- `translate.py --skip-existing` пропускает уже переведённые страницы
- Новые языки переводятся с нуля
- hreflang и lang-switcher обновляются с полным списком

## Неанглийские сайты

- `detect_source_lang()` читает `<html lang="">` и автоматически определяет язык
- Исходный язык **исключается** из целевых переводов
- Арабские сайты: `dir="rtl"` добавляется в языковые версии; LTR-переводы арабского источника теряют rtl
- SEO-фиксы работают независимо от языка

> ⚠️ Groq генерирует descriptions и H1/H2 на английском. Для неанглийских источников текст создаётся через EN-посредника, после чего переводится. На практике результат приемлемый.

## Pull Request

- Клонирует репо через `git clone`, копирует обработанные файлы, делает `git push`
- Создаёт ветку `seo-fixes-YYYYMMDD-HHMM`
- Нет ограничений на размер файлового дерева
- Приоритет токена: пользовательский `ghp_...` → системный `GITHUB_TOKEN`

## Локальный запуск (run_local.py)

Для обработки сайта без бота и GitHub:

```bash
# Один сайт
python run_local.py --site d:/loricarson --domain loricarson.com

# Только SEO, без перевода
python run_local.py --site d:/mysite --domain mysite.com --mode seo_only

# Только перевод (SEO уже сделан)
python run_local.py --site d:/mysite --domain mysite.com --mode translate --langs ru,de,fr

# Все сайты из списка SITES
python run_local.py --all
```

Переменные окружения: `GROQ_KEY`, `WOWAI_KEY`.
