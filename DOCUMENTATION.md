# SEO Bot — Документация

**Версия:** v2 (апрель 2026)  
**Деплой:** Railway (auto-deploy из GitHub `master`)

---

## Что такое SEO Bot

Telegram-бот для автоматической SEO-оптимизации статических сайтов на GitHub. Принимает ссылку на репозиторий, проводит аудит, применяет исправления, переводит сайт на до 21 языка и открывает Pull Request — без ручного редактирования кода.

**Целевая аудитория:** владельцы сайтов, сохранённых через Wayback Machine, без хостинга и технического бэкграунда.

---

## Архитектура

```
bot.py          — Telegram UI, FSM-состояния, оркестрация pipeline
detector.py     — Определяет тип репо, извлекает сайт из web.archive.org
audit.py        — SEO-аудит: сканирует HTML, формирует список проблем
fixes.py        — Шаги исправлений (in-place)
translate.py    — Перевод через WowAI API, hreflang, языковой switcher
```

**Внешние зависимости:**
- **WowAI Translate API** — машинный перевод (21 язык)
- **Groq API** (llama-3.1-8b-instant) — генерация meta description
- **GitHub API + git** — скачивание репо и создание Pull Request
- **aiogram 3** — Telegram Bot Framework
- **Railway** — хостинг бота

## Поддерживаемые форматы репозитория

| Тип | Описание |
|---|---|
| `web.archive.org/` дамп | ZIP из Wayback Machine — бот извлекает HTML/CSS/JS, очищает от артефактов |
| `site/` папка | Уже извлечённый сайт |
| HTML в корне | Статические файлы напрямую в корне репозитория |

**Приоритет детектора:**
1. `site/` с HTML → использует как есть
2. `web.archive.org/` → извлекает в `site/`
3. HTML в корне → работает с корнем напрямую

**Повторный прогон:** если в репо уже есть языковые папки (`/de/`, `/ru/`...) — бот копирует их в рабочую директорию и пропускает уже переведённые страницы.

## Режимы работы

| Режим | Кнопка | Описание |
|---|---|---|
| Аудит | 🔍 Только аудит | Сканирует репо, выводит список SEO-проблем. Ничего не меняет. |
| Полный | 🔧 Полный (SEO + перевод + PR) | Все шаги: SEO-фиксы + перевод + Pull Request |
| SEO без перевода | 🛠️ SEO без перевода + PR | SEO-фиксы (meta, canonical, schema, og:image...) без перевода |
| Только перевод | 🌍 Только перевод + PR | Пропускает SEO-шаги, только переводит + PR. Для повторного прогона с новыми языками. |

```
Отправить ссылку: github.com/user/sitename
        │
        ▼
  Выбор режима
  ├── 🔍 Только аудит
  │     └── Домен → Выбор языков → Аудит → Готово
  │
  ├── 🛠️ SEO без перевода
  │     └── Домен → GitHub токен → Аудит → Подтверждение → SEO-фиксы → PR
  │
  ├── 🔧 Полный
  │     └── Домен → GitHub токен → Выбор языков → Аудит → Подтверждение → Фиксы + Перевод → PR
  │
  └── 🌍 Только перевод
        └── Домен → GitHub токен → Выбор языков → Аудит → Подтверждение → Перевод → PR
```

**Выбор языков:** после токена появляется два варианта:
- `🌍 Все языки (21)` — мгновенно стартует со всеми языками
- `✏️ Выбрать вручную` — открывает сетку с флагами, кнопки "Выбрать все" / "Снять все"

**Оценка времени** показывается до подтверждения:
```
⏱ Примерное время обработки: ~320–450 мин (68 стр. × 21 яз., 66 с контентом)
```

**Живой прогресс** во время перевода:
```
🌍 Перевод: 23/66 страниц...
```

**Аудит до/после** — после фиксов бот показывает дельту:
```
📊 Аудит до/после:
Проблем до: 47 → после: 12 (−35 ✅)
```

## Шаги исправлений

### Полный режим (SEO + перевод)

| # | Шаг | Что делает |
|---|---|---|
| 1 | `fix_archive_scripts` | Удаляет Wayback Toolbar инъекции из HTML |
| 2 | `fix_canonical` | Добавляет абсолютный `<link rel="canonical">` с реальным доменом |
| 3 | `fix_title_refresh` | Обновляет год в title/description на текущий |
| 4 | `fix_descriptions` | Генерирует уникальный meta description через Groq AI (120–155 символов) |
| 5 | `fix_schema` | Добавляет BreadcrumbList Schema.org |
| 6 | `fix_og_image` | Добавляет og:image из первого изображения страницы |
| 7 | `fix_nofollow` | Добавляет `rel="nofollow noopener noreferrer"` на внешние ссылки |
| 8 | `fix_robots_txt` | Создаёт/обновляет robots.txt со ссылкой на sitemap |
| 9 | `fix_translations` | Переводит на выбранные языки (с кэшем nav/footer) |
| 10 | `fix_hreflang_translated` | Добавляет hreflang на переведённые страницы |
| 11 | `fix_internal_links` | Исправляет внутренние ссылки в переведённых версиях |
| 12 | `fix_lang_switcher` | Добавляет/обновляет floating dropdown для переключения языков |

### SEO без перевода — шаги 1–8 (без 9–12)

### Только перевод — шаги 9, 10, 12 (без SEO-шагов и fix_internal_links)

## Перевод: как работает

1. **Детектирование языка** сайта из `<html lang="">` — поддержка неанглийских источников
2. **Кэш nav/footer** — UI-строки (ссылки меню, кнопки) переводятся 1 раз на сессию (~30% экономии API)
3. **Расширенное извлечение**: title, description, H1–H4, p, li, button, `<span>`, `<figcaption>`, `<input placeholder>`, `.card-title`, `.card-text`
   - Заголовки с вложенными тегами (`<br/>`, `<span>`) разбиваются на отдельные текстовые узлы — каждый переводится и заменяется независимо
   - Многострочные `<p>` (с переносами строк внутри тега) корректно распознаются и заменяются
4. **Батч-перевод** через WowAI API (по 50 сегментов на запрос)
5. **Антифейк-фильтр**: перевод отклоняется если ≥70% совпадает с оригиналом (для кириллических/иероглифических языков — character ratio). ⚠️ Украинский (`uk`) в WowAI возвращает исходный английский текст — известная проблема, в бэклоге
6. **RTL поддержка**: арабский получает `dir="rtl"`, LTR-переводы арабского источника теряют rtl
7. **Запись**: переведённый файл → `/{lang}/path/index.html`
8. **hreflang** мержится в оригинальный файл — не перезаписывает прошлые прогоны

### Повторный запуск (incremental)

Если репозиторий уже был обработан ботом (PR смёрджен), при повторном запуске:
- Детектор распознаёт существующие языковые папки (`/ru/`, `/ko/` и т.д.) в корне репо — использует корень как site_dir, **не переизвлекает из архива**
- `translate.py --skip-existing` пропускает уже переведённые страницы
- Новые языки переводятся с нуля
- hreflang обновляется с полным списком (старые + новые языки)
- Lang-switcher перестраивается со всеми доступными языками

## Очередь репозиториев

Бот обрабатывает до 3 задач одновременно (1 активная + 2 в ожидании):
- Новая задача пока бот занят → ставится в очередь с уведомлением о позиции
- После завершения активной → автоматически запускается следующая с уведомлением
- Один пользователь не может поставить в очередь дважды

## Pull Request

- Клонирует репо через `git clone`, копирует обработанные файлы, делает `git push`
- Создаёт ветку `seo-fixes-YYYYMMDD-HHMM`
- Нет ограничений на размер дерева файлов (убрано Git API с лимитом 546 файлов)
- Приоритет токена: пользовательский `ghp_...` → системный `GITHUB_TOKEN`

## Как запустить новый сайт

1. Сохранить сайт через [web.archive.org](https://web.archive.org) → **Save Page Now**
2. Скачать архив: Browse History → **Download ZIP**
3. Создать GitHub репозиторий, загрузить ZIP
4. Отправить боту ссылку: `https://github.com/user/repo`
5. Выбрать режим, указать домен, выбрать языки
6. Дождаться Pull Request (время показывается до старта)
7. Смёрджить Pull Request
